# SOP Akun Quotes X — Indeks Utama

> Indeks utama seluruh dokumen SOP pengelolaan Akun_Quotes di Platform_X. Dokumen dikelompokkan menjadi dua domain sejajar: **operasional konten** dan **operasional bisnis**.

## Struktur

```
sop/
├─ README.md                  # indeks utama (dokumen ini)
├─ operasional-konten/        # SOP konten & akun (dokumen master + lampiran)
│  ├─ SOP-Akun-Quotes-X.md    # Dokumen master (Bagian 0–11)
│  ├─ lampiran-template.md    # Template postingan, klarifikasi, log pembelajaran
│  ├─ lampiran-checklist.md   # Checklist pra-posting, moderasi, keamanan
│  ├─ lampiran-contoh.md      # Contoh format postingan, thread, & balasan
│  └─ lampiran-monetisasi-x.md# Peraturan boleh & tidak boleh monetisasi (acuan Content Monetization Standards X)
└─ operasional-bisnis/        # Ketentuan bisnis & finansial antara Founder dan Co-Founder
   ├─ README.md               # Indeks folder bisnis
   ├─ modal-dan-bagi-hasil.md # Modal awal, dana cadangan, sistem bagi hasil (SOP-B.1–B.5)
   └─ ketentuan-bisnis.md     # Kepemilikan, peran, pengeluaran, pencatatan, tata kelola (SOP-B.6–B.10)
```

## Dua Domain

- **`operasional-konten/`** — pedoman operasional harian konten dan akun: kriteria konten & Quotes, verifikasi sumber & atribusi, styleguide, following, interaksi/komentar, moderasi, eskalasi krisis, jadwal & kalender konten, metrik & kelayakan monetisasi, dan keamanan akun. Dokumen master `SOP-Akun-Quotes-X.md` adalah sumber utama; lampiran memuat template, checklist, contoh, dan aturan monetisasi.
- **`operasional-bisnis/`** — kesepakatan bisnis/finansial antara Founder dan Co-Founder: modal awal, dana cadangan langganan, sistem bagi hasil (waterfall), kepemilikan, peran, pengeluaran, pencatatan keuangan, dan tata kelola.

## Konvensi Bersama

- ID aturan stabil: `SOP-N.x` (konten), `SOP-M.x` (lampiran monetisasi), `SOP-B.x` (bisnis).
- Anotasi keterlacakan `(Requirement R{n}.{m})` pada setiap aturan.
- Penanda ketidaklengkapan `> [TODO: ...]` pada bagian yang belum lengkap (R1.5).
- Kontrol dokumen & versi diatur pada Bagian 0 dokumen master.
